AV女優ページ構造・設計ドキュメント（日本語）

本ドキュメントは、AVFZK サイトにおける
AV女優個別ページおよび一覧ページの構造設計について説明する技術資料です。

対象サイト
https://www.avfzk.com/

1. AV女優ページの目的

AV女優ページは、以下の目的で設計されています。

女優名を軸にした情報集約

出勤情報・在籍店舗との関連付け

ジャンル・エリアとの内部リンク形成

検索エンジン向けの構造化データ提供

本ページは、検索エンジンとユーザー双方にとって
理解しやすく、再利用可能な構造を重視しています。

2. 女優個別ページのURL構造

AV女優の個別ページは、以下の形式を採用しています。

/actress/{encoded-name}


特徴：

クエリパラメータを使用しない

英数字ベースの一意な識別子

将来的なURL変更リスクを低減

3. ページ内の基本構成

各女優ページは、以下の情報ブロックで構成されています。

女優名（日本語）

読み仮名

プロフィール画像

基本情報（年齢・出身地など）

在籍店舗一覧

出勤エリア情報

関連ジャンルリンク

これにより、単一ページ内で
検索意図の完結を目指しています。

4. 一覧ページとの関係

女優個別ページは、以下の一覧ページと相互にリンクされています。

女優一覧ページ
/girllist/

ジャンル別ページ
/genre/

エリア別ページ
/area/

これにより、サイト全体の内部回遊性を高めています。

5. 構造化データ（JSON-LD）

女優ページでは、Schema.org の Person タイプを使用した
構造化データを埋め込んでいます。

主な項目：

name

alternateName

image

url

一覧ページでは ItemList として定義し、
各女優を Person として列挙します。

6. SEO上の設計意図

本構造は、以下の点を意識して設計されています。

重複コンテンツの回避

特定のキーワードに依存せず、
構造そのものの評価を重視しています。
